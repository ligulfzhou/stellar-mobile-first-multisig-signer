# Vault Signer

Mobile-first multisig treasury signer for **Stellar Soroban**. Rust core, UniFFI bindings, iOS/Android demo apps.

Soroban **vault contract** (C-address treasury) with N-of-M signers — not classic G-account multisig. The app connects to an existing vault, lists pending proposals, and signs approve/reject; it can also deploy new vaults via the factory.

**Status:** working testnet demo (CLI + E2E + iOS). Development paused pending the next Stellar grant round.

## What's in the demo

| Layer | Done |
|-------|------|
| Contracts | `multisig-vault`, factory, registry |
| Rust | keys, RPC, vault read/write, factory `create_vault` |
| CLI | config, signers, proposals, propose, approve, reject, execute |
| Testnet E2E | fund → propose → approve → execute |
| iOS | Vault / Create / Settings tabs, Keychain, async RPC |
| Android | Compose shell (parity with iOS) |

## Prerequisites

- Rust (nightly for `just build` / fmt)
- [Stellar CLI](https://developers.stellar.org/docs/tools/cli) with funded testnet identity (`admin`, optionally `alice`)
- iOS: Xcode 16+, [xcodegen](https://github.com/yonaskolb/XcodeGen) (`brew install xcodegen`)
- Android: Android Studio, NDK (for `just android-lib`)

## Build

```bash
just check          # or: just build
just contract-build # Soroban WASM
just ffi            # UniFFI + bindings/swift, bindings/kotlin
```

## CLI reference

```bash
export VAULT_ADDRESS=C...
export SOROBAN_RPC_URL=https://soroban-rpc.testnet.stellar.gateway.fm
export STELLAR_SECRET=S...   # for write commands

cargo run -p vault-signer-cli -- config
cargo run -p vault-signer-cli -- proposals --pending-only
cargo run -p vault-signer-cli -- proposal --id 1

# Write (needs --secret or STELLAR_SECRET)
cargo run -p vault-signer-cli -- propose --recipient G... --amount 1000000 --secret "$STELLAR_SECRET"
cargo run -p vault-signer-cli -- approve --id 1 --secret "$STELLAR_SECRET"
cargo run -p vault-signer-cli -- execute --id 1 --recipient G... --amount 1000000 --secret "$STELLAR_SECRET"
cargo run -p vault-signer-cli -- reject --id 1 --secret "$STELLAR_SECRET"
```

## Mobile apps

### iOS

```bash
just ios-setup    # bindings + static libs + VaultSigner.xcodeproj
open ios/VaultSigner.xcodeproj
```

Run on Simulator (e.g. iPhone 17). After Rust API changes, run `just ios-setup` again.

More: [ios/README.md](ios/README.md)

### Android

```bash
just android-setup   # bindings + libvault_signer_ffi.so (arm64)
```

Open `android/` in Android Studio. Requires NDK; see [android/README.md](android/README.md).

### App flow

1. **Settings** — import signer secret (Keychain / EncryptedSharedPreferences), confirm vault/factory/RPC
2. **Vault** — dashboard, pending proposals, approve/reject
3. **Create** — deploy new N-of-M vault via factory (creator must be in signer list, account needs XLM for fees)
